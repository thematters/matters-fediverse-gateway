# Decision 03 · 付費文外部呈現策略

狀態：**已定 · A · 完全隱形**
拍板人：mashbean（總經理）
拍板日期：2026-04-25
影響範圍：W5（付費/私密邊界程式化）動工前必須定案；牽涉法務與商業
最後更新：2026-04-25

---

## 決議

**選 A · 完全隱形**：付費文 / 加密文 / 私訊不產生任何 ActivityPub 活動，外部聯邦完全看不到這篇存在（連標題都不出現）。

**理由**：
- 邊界最清楚、零洩漏風險、法務最安全
- 商業模式繫於訂閱經濟；任何形式的「有出現」都可能被解讀為部分洩漏，不值得為了引流冒險
- 與加密文邊界一致，不需做雙套邏輯

**實作衝擊**：
- W5 的 visibility gate 邏輯簡化：只要 visibility ≠ public 就 drop，不做 preview / 標題卡分支
- `static-outbox-bridge` 直接過濾掉 non-public 文章，不會出現在 outbox
- `Update` / `Delete` 也不對非公開文章 fan-out（避免被外部用差異推論存在）
- 整合測試斷言：偽造 paid / encrypted / private 文章經 bridge 後，outbox 與 Create activity 完全不出現

---

## 問題

當作者有付費訂閱 / 解鎖內容時，這篇文章在聯邦上要怎麼呈現？

完全不出現？只出現標題與 canonical link？出現幾段 preview 後切斷？

---

## 選項

### A · 完全隱形
付費文不產生任何 ActivityPub 活動，外部聯邦看不到這篇存在。

**優**：邊界最清楚、零洩漏風險、法務最安全。
**劣**：作者錯失導流機會；訂閱者也看不到自己訂閱的作者「有發新文」這個訊號。

### B · 標題卡（只出現「有新文」）
產生一個 `Article` 但 `content` 只有「（付費內容）」+ canonical link，`name` 帶標題，`summary` 留空。

**優**：作者可被外部讀者「看到他在持續創作」；引流回 Matters；不洩漏內容。
**劣**：聯邦上作者的時間軸看起來都是「（付費內容）+ 連結」，閱讀體驗單調；無法說服讀者點進來。

### C · 限額 preview（前 N 字 / 前 N 段）
帶出文章前 N 字（建議 200–400 字或第一段），標明「（節錄）」，附 canonical link 引流。

**優**：保留 hook 的吸引力；引流力最強；近似 Substack 模式，使用者熟悉。
**劣**：「N 字」邊界要法務 + 產品共定；多語斷字策略要寫清楚；作者可選擇 opt-out。

### D · 作者選擇（per-article）
編輯器加一個欄位讓作者自選 A/B/C，預設 B。

**優**：尊重作者自主；每篇可細調。
**劣**：UI 多一個欄位，作者體驗變重；預設值的選擇仍要團隊定。

---

## 法務與商業考量

- 訂閱經濟邏輯不能被 preview 破壞：N 字若太多形同免費。
- GDPR / 個資：preview 內容不能含付費讀者資料、評論區等。
- 法律下架：付費文被檢舉時的下架流程要與公開文一致。
- 中港讀者隔離：付費文 preview 是否也走牆內隔離？這點與 Matters 既有原則一致即可。

---

## 建議

**首發選 B（標題卡），G2 階段視作者回饋升級到 D（讓作者選 B/C）。**

理由：
- B 是邊界最清楚的「有出現但不洩漏」方案，首發風險最低
- C 的 N 字邊界需要法務、產品、多語都要對齊，不適合首發節奏
- D 把選擇權交回作者是長期方向，但需要前端 UI 配合，不在 G1 範圍

## 待 mashbean 拍板

- 採 A / B / C / D 哪個作為首發？
- 若選 B，標題卡的固定文案怎麼寫？建議「{title} —— 本篇為訂閱作者付費內容，可至 matters.town 閱讀」
- 加密文（讀者解密的私密文）邊界與付費文是否一致？建議完全隱形（A），因為加密文連標題都不應外流
