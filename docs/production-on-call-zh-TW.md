# Matters Fediverse 正式監控與客服 on-call

## 文件用途

本文件供上線值班、客服、營運與工程人員使用。正式開放前，必須填入以下負責人。

- 上線負責人 `[待填]`
- 工程 on-call `[待填]`
- 客服 on-call `[待填]`
- 產品決策人 `[待填]`
- 值班 Slack 頻道 `[待填]`

監控告警統一送到既有 production Slack 告警管道。`/next/fediverse` 是人工查詢、重送與結案介面。

## 監控範圍

正式告警包含以下項目。

- Gateway 五分鐘心跳中斷
- Gateway 出現未結案死信
- Gateway 最早待投遞項目超過五分鐘
- SQS 主佇列最早訊息超過五分鐘
- SQS 死信佇列出現訊息
- `federation-export-prod` Lambda 出現錯誤或節流
- Gateway EC2 狀態檢查失敗

上線後前四小時，每十五分鐘人工查看一次以下數字。

- SQS 主佇列可見、處理中與延遲訊息
- SQS 死信佇列訊息
- Gateway pending、retry pending 與 dead letter
- Lambda 錯誤與節流
- `/next/fediverse` 最新事件與錯誤原因
- 一篇新文章、一次修改與一次封存的遠端實讀結果

## 事件等級

### P0 立即停止新投遞

符合任一條件即為 P0。

- 付費、圍爐、草稿、封存或其他非公開內容送到遠端
- ActivityPub 私鑰、操作 token 或資料庫憑證疑似外洩
- 大量作者身分錯置，文章由錯誤帳號送出

處理目標為五分鐘內確認，十五分鐘內停止新投遞。

### P1 重大投遞故障

符合任一條件即為 P1。

- Gateway 心跳中斷
- SQS 或 Gateway 最早待投遞時間超過五分鐘
- 死信佇列或 Gateway 死信出現未結案項目
- Lambda 持續錯誤或節流
- 多個遠端服務都無法收到新文章

處理目標為十五分鐘內確認，三十分鐘內完成緩解或回滾決策。

### P2 單一服務或單一作者問題

常見情況如下。

- 只有 Threads 或單一遠端服務延遲
- 單一作者搜尋不到帳號
- 單篇文章未顯示、未更新或未消失
- 遠端排版、圖片或摘要呈現不同

處理目標為當班完成初查，保留可追蹤案件並在一個工作日內更新。

## 告警收到後的前十五分鐘

1. 在 Slack 告警串回覆「已接手」、時間與值班人
2. 開啟 `/next/fediverse`，記錄 pending、retry pending、dead letter 與最早待投遞時間
3. 查看 SQS 主佇列與死信佇列
4. 查看 `federation-export-prod` 最近錯誤
5. 檢查 Gateway 健康狀態與 EC2 狀態
6. 判斷是否有內容邊界問題
7. 在告警串更新事件等級、影響範圍與下一次更新時間

若涉及內容邊界、憑證或身分錯置，立即依 P0 回滾，不等待完整根因。

## 回滾

1. 將 `MATTERS_FEDERATION_EXPORT_TRIGGER_MODE` 設為 `off`
2. 停用 `federation-export-prod` 的 SQS event source mapping，保留佇列內容
3. 若 ActivityPub 公開路由異常，關閉 Worker 的一般作者路由
4. 保留 Gateway、`/next/fediverse`、SQS、DLQ、稽核紀錄、actor registry 與簽章金鑰供調查
5. 在 Slack 公告影響範圍、回滾時間與下一次更新時間

回滾期間不得清空 SQS、DLQ 或 Gateway 死信，也不得用重新發文代替重送。

## 死信處理

每一筆死信只能採以下其中一種處理。

- 修正暫時性問題後重送，填寫原因與操作人
- 確認遠端永久拒絕後結案，填寫接收端、狀態碼與判斷依據
- 涉及安全或內容邊界時保持封存，交由產品與安全負責人決定

重送前先確認 Activity ID、文章 ID、作者、接收端與冪等鍵。重送後必須確認佇列、Gateway 狀態與遠端實讀。

## 客服收件欄位

客服請一次收齊以下資訊。

- Matters 帳號
- 聯邦帳號 `@帳號@matters.town`
- Matters 文章網址
- 發佈、修改或封存時間
- 目標服務名稱
- 問題類型
- 畫面截圖
- 是否已等待十五分鐘

客服不得要求使用者重新發文，也不得承諾遠端副本會立即或完整消失。

## 客服回覆範本

### 遠端尚未顯示

我們已收到你的聯邦發佈問題。Fediverse 內容會由目標服務自行接收與建立索引，因此可能有短暫延遲。我們會依你提供的文章網址、發佈時間與目標服務查詢投遞狀態，請先不要重複發佈同一篇文章。

### 修改尚未同步

Matters 已在文章修改後建立聯邦更新事件。遠端服務可能保留快取並稍後才顯示新版內容。我們會確認更新事件是否已成功送達，並在查核後回覆。

### 刪除後仍看得到

Matters 會在文章封存、關閉個別文章聯邦發佈或關閉作者聯邦發佈時送出刪除請求。Fediverse 由獨立服務組成，各服務可能保留快取、備份或轉發副本，因此無法保證所有遠端內容立即或完整移除。我們會確認 Matters 的刪除事件是否已送達目標服務。

### Threads 搜尋不到帳號

請先確認搜尋格式為 `@你的帳號@matters.town`，並確認已開啟 Fediverse 聯邦發佈且發佈過新的公開文章。Threads 可能需要時間建立遠端帳號資料。我們會依帳號、文章網址、搜尋時間與截圖協助查核。

## 交班

每次交班必須留下以下內容。

- 目前事件等級與影響範圍
- 最近一次人工檢查時間
- SQS 與 Gateway 佇列數字
- 尚未結案死信
- 已執行的重送、結案或回滾操作
- 下一個明確動作與負責人
- 下一次對內與對外更新時間

正式開放前，需由上線負責人確認負責人欄位已填妥、Slack 告警測試已收到、客服範本已交付。
